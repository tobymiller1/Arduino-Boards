12 If you ever wanted a key that works like keys on old cell phones, when you press
13 a key and it cycles through a number of options in a sequence, then the cycling
14 key is what you are looking for. It is a bit different than on cell phones of
15 old, as it is a separate key, that works in combination of other keys: you press
16 a key, then the cycle key, and the cycle key will replace the previously input
17 symbol with another. Keep tapping the cycle key, and it will replace symbols
18 with new ones, in a loop.
19 
20 ## Using the plugin
21 
22 To use the plugin, we need to include the header, and declare the behaviour
23 used. Then, we need to place a cycle key or two on the keymap. And finally, we
24 need to implement the [`cycleAction`][cycleaction] function that gets called
25 each time the cycling key triggers.
26 
27  [cycleaction]: #cycleactionpreviouskey-cyclecount
28 
29 ```c++
30 #include <Kaleidoscope-Cycle.h>
31 
32 // Somewhere in the keymap:
33 Key_Cycle
34 
35 // later in the Sketch:
36 void cycleAction(Key previous_key, uint8_t cycle_count) {
37  if (previous_key.raw == Key_A.raw) {
38  cycleThrough (Key_B, Key_C, Key_D);
39  }
40 }
41 
42 void setup(void) {
43  Kaleidoscope.use(&Cycle);
44 
45  Kaleidoscope.setup();
46 }
47 ```

49 ## Keymap markup
50 
51 ### `Key_Cycle`
52 
53 > The key code for the cycle key. There can be as many of this on the keymap, as
54 > many one wants, but they all behave the same. There is little point in having
55 > more than one on each side.

57 ## Plugin methods
58 
59 The plugin provides a `Cycle` object, but to implement the actions, we need to
60 define a function ([`cycleAction`][cycleaction]) outside of the object. A
61 handler, of sorts. The object also provides a helper method to replace the
62 previous symbol with another. The plugin also provides one macro that is
63 particularly useful, and in most cases, should be used over the `.replace()`
64 method explained below.
65 
66 ### `cycleThrough(keys...)`
67 
68 > Cycles through all the possibilities given in `keys` (starting from the
69 > beginning once it reached the end). This should be used from
70 > the [`cycleAction`][cycleaction] function, once it is determined what sequence
71 > to cycle through.
72 >
73 > To make the cycling loop complete, the first element of the `keys` list should
74 > be the one that - when followed by the Cycle key - triggers the action.
75 
76 ### `.replace(key)`
77 
78 > Deletes the previous symbol (by sending a `Backspace`), and inputs the new
79 > one. This is used by `cycleThrough()` above, behind the scenes.
80 >
81 > The recommended method is to use the macro, but in special circumstances, this
82 > function can be of direct use as well.
83 
84 ## Overrideable methods
85 
86 ### `cycleAction(previous_key, cycle_count)`
87 
88 > The heart and soul of the plugin, that must be defined in the Sketch. It will
89 > be called whenever the cycling key triggers, and the two arguments are the
90 > last key pressed (not counting repeated taps of the cycling key itself), and
91 > the number of times the cycling key has been pressed.
92 >
93 > It is up to us to decide what to do, and when. But the most common - and
94 > expected - action is to call `cycleThrough()` with a different sequence for
95 > each key we want to use together with the cycling key.
